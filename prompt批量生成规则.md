# AI人像批量生成规则
生成日期：2026-06-04

文件定位：prompt 生成规则。

用途：根据 `AI人像风格测试分类.md` 中的风格分类，批量生成可用于图像生成的中文 prompt。本文件只负责生成 prompt，不直接调用 Codex Imagen 生成图片。

用途：作为批量生成 AI 人像测试图的执行规则。每个场景都要分别生成 `女性.jpg` 和 `男性.jpg` 两个参考图版本，并保持同组图片为同一个地点或同一个核心场景。

## 一、生成顺序

1. 每个场景依次生成两张图。
2. 第一张使用 `女性.jpg` 作为参考图。
3. 第二张使用 `男性.jpg` 作为参考图。
4. 两张图必须围绕同一个地点、景区、街区、建筑、自然环境、节日现场或室内空间展开。

## 二、同场景一致性

1. 同一组两张图必须描述同一个地点或场景，例如洛阳牡丹园、京都樱花街巷、巴黎咖啡馆街头、三亚海边日落等。
2. 背景核心元素要大致一致，包括地标、建筑、自然景观、节日装饰、街景层次或室内陈设。
3. 服饰风格、配色、光影和镜头质感可以不同，但必须服务于同一个场景，不能变成另一个主题。
4. 人物位于画面正中央，正面看向镜头，约占画面 60%。
5. 背景色彩与人物服饰色彩要形成清晰对比，整体保持高饱和、明亮、有层次。

## 三、两个参考图版本差异

1. `女性.jpg` 版本可以更明快、柔和、华丽或浪漫，常用胭脂粉、朱红、孔雀蓝、玉绿、金黄、淡樱粉等高饱和配色。
2. `男性.jpg` 版本需要更沉稳、有质感，常用深蓝、墨绿、酒红、棕褐、焦糖、铜金、象牙白等高饱和但稳定的主色，并加入少量亮色点缀。
3. 历史服饰、民族服饰、海外传统服饰等场景必须符合当地文化和场景审美，不能出现地点与服装文化错配。
4. 提示词正文不要主动写入“男性”“女性”“男士”“女士”等性别标签，统一使用“参考人物”“人物”“主体”等中性表达；气质差异通过配色、服饰剪裁、配饰和光影体现。

## 四、服饰文化适配要求

1. 服饰描述必须结合当地特色、景点文化、历史背景或地域审美，写出具体可识别的服饰名称或服饰体系。
2. 不要只写“旅拍服饰”“旅拍造型”“古风服饰”“精致穿搭”等泛化描述。
3. 可以使用当地民族服饰、历史服饰、非遗服饰、城市文旅改良服饰、景区主题服饰等具体表达，例如苗族银饰盛装、侗族织锦服饰、屯堡服饰、唐制襦裙、明制圆领袍、宋制褙子、藏式氆氇服饰、江南旗袍、岭南香云纱、海岛黎锦服饰等。
4. 现代旅拍场景也要写出与当地文化适配的服饰名称，例如“融合苗绣纹样的现代短外套”“带蜡染纹样的城市旅拍套装”“参考青岩古镇石雕纹样的改良长衫”。
5. 同一场景的两个参考图版本可以使用同一文化体系下的不同剪裁、配色和配饰，但都必须能看出与场景或景点文化有关。
6. 服饰名称必须自然写入提示词正文，和地点、建筑、节日、街景或自然环境形成统一画面。

## 五、提示词关键要求

1. 每条提示词必须用中文单段书写。
2. 以“参考图片中人物的五官，生成xxxx”作为提示词开头，其中 `xxxx` 替换为本次要生成的具体场景、景区、风格或主题。
3. 提示词不要写入图片尺寸字样，例如 `1024x1024`、`1024*1024`、`方图`。
4. 提示词正文不要写否定式约束、反向提示词或“避免/不要/不能/无/禁止”开头的排除项，需要控制画面时改用正向描述。
5. 每条提示词都要符合本文件的统一规范，并参考 `AI人像风格测试分类.md` 选择风格类别。

## 六、prompt 输出与后续调用关系

1. 本文件输出的是“生成好的 prompt”，不是最终图片。
2. 每次生成 prompt 时，必须在 `outputs/prompts` 下创建一个带时间的文件夹。
3. 生成好的男女 prompt 必须写入该时间文件夹中，作为 `批量图像生成规则.md` 的输入。
4. `批量图像生成规则.md` 只能调用已经生成好的 prompt，不在图像生成阶段重新改写 prompt。

## 七、prompt 文件夹规则

1. 每次执行 prompt 批量生成时，新建一个时间文件夹，目录格式建议为：

```text
outputs/prompts/YYYYMMDD_HHMMSS
```

示例：

```text
outputs/prompts/20260605_232500
```

2. 同一批次的所有 prompt 都保存到该时间文件夹内。
3. 每个场景必须分别保存 `女性.jpg` 参考图版本和 `男性.jpg` 参考图版本的 prompt。
4. 文件名建议使用以下格式：

```text
{场景编号}_{场景名称}_女性参考图_prompt.txt
{场景编号}_{场景名称}_男性参考图_prompt.txt
```

示例：

```text
01_洛阳牡丹唐朝古装旅拍_女性参考图_prompt.txt
01_洛阳牡丹唐朝古装旅拍_男性参考图_prompt.txt
```

5. prompt 文件内容只写最终可调用的 prompt 正文，不写解释、分析、尺寸参数或图像生成参数。

## 八、单场景输出格式

场景编号：例如 `01`

场景名称：例如 `洛阳牡丹唐朝古装旅拍`

共用场景设定：记录两张图共同使用的地点、地标背景、文化来源、核心场景元素、适配服饰文化名称和画面主体要求。

`女性.jpg` 版本：在共用场景设定基础上生成一条完整提示词，必须写出适配场景或景点文化的具体服饰名称，色彩和光影可更明快精致。

`男性.jpg` 版本：在共用场景设定基础上生成一条完整提示词，地点和场景保持一致，必须写出适配场景或景点文化的具体服饰名称，色彩和光影可更沉稳有质感。

## 九、生成检查

1. 是否先生成 `女性.jpg` 版本，再生成 `男性.jpg` 版本。
2. 两条提示词是否保持同一地点或同一场景。
3. 背景核心元素是否大致一致。
4. 构图是否为人物居中、正面看镜头、约占画面 60%。
5. 两个版本是否有气质差异，但都保持色彩丰富、文化准确、画面有层次。
6. 提示词是否保留固定开头句式，并且没有写入性别标签、尺寸字样和否定式约束。
7. 是否已将男女 prompt 写入 `outputs/prompts` 下最新创建的时间文件夹。
8. 服饰描述是否写出了适配当地特色、场景或景点文化的具体名称，而不是泛化的旅拍服饰。

## 十、命名建议

`场景编号_场景名称_女性参考图.png`

`场景编号_场景名称_男性参考图.png`

例如：

`01_洛阳牡丹唐朝古装旅拍_女性参考图.png`

`01_洛阳牡丹唐朝古装旅拍_男性参考图.png`
